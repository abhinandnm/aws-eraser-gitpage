Why AWS Eraser?

Built for Speed, Safety, and Maximum Savings

Avoid unexpected AWS charges caused by forgotten sandbox instances, idle NAT gateways, and unattached EBS volumes.

Multi-Region Scanning

Automatically iterates across all 25+ global AWS regions (including opt-in regions) to locate hidden resources draining your wallet.

Selective or Complete Nuke

Choose between nuking all resources at once or interactively selecting specific item numbers or ranges (1, 3, 5-8).

Zero Cloud Footprint

Runs entirely on your local machine using standard AWS Python SDK (boto3). No CloudFormation templates or agent deployments needed.

Smart EBS Wait Handler

Detects EBS volumes locked in VolumeInUse status attached to terminating instances and automatically polls and purges them.

S3 Bucket & Version Purger

Recursively deletes all objects, historical object versions, and delete markers before deleting non-empty S3 buckets seamlessly.

Cost Explorer Integration

Fetches your current accrued monthly bill by service so you can review cost drainers before confirming deletion.

What AWS Eraser Cleans vs Preserves

Designed to eliminate costly infrastructure without revoking your credentials or breaking non-deletable safety barriers.

Cleaned

EC2 Instances

Terminates running and stopped virtual machines across all regions.

Cleaned

Lightsail Instances

Deletes Lightsail compute VPS instances and attached disks.

Cleaned

S3 Buckets & Versions

Empties all current objects, versions, and delete markers before purging bucket.

Cleaned

EBS Volumes & Snapshots

Deletes unattached or locked volumes and user-created EC2 EBS volume snapshots.

Cleaned

RDS Databases & Backups

Deletes RDS database instances, manual snapshots, and automated backup retention.

Cleaned

DynamoDB Tables

Deletes DynamoDB NoSQL tables and indexes.

Cleaned

NAT Gateways & EIPs

Deletes hourly NAT Gateways and releases idle Elastic IP addresses.

Cleaned

Load Balancers (ALB/NLB)

Deletes Application, Network, and Classic Load Balancers.

Cleaned

WAFv2 Web ACLs

Removes Global (CloudFront) and Regional Web Application Firewall ACL rules.

Preserved

IAM Users & Credentials

Preserved so the active credentials running the script are never locked out.

Preserved

KMS Encryption Keys

KMS keys require an irreversible 7 to 30 day AWS wait period and are preserved.

Preserved

Savings Plans & Contracts

Financial contracts cannot be deleted via AWS API and must be managed manually.

Instant Execution

How to Run AWS Eraser

Execute directly in your terminal in under 60 seconds.

PowerShell / Command Prompt (Auto-installs dependencies) 
# Download and execute run.bat in your repository directory:
.\run.bat
Terminal (requires boto3) 
# Install boto3 dependency:
pip install boto3

# Launch AWS Eraser interactive CLI:
python aws_eraser.py
Git Repository Setup 
git clone https://github.com/abhinandnm/aws-eraser.git
cd aws-eraser
pip install -r requirements.txt
python aws_eraser.py

No local credentials file? The script safely prompts for temporary AWS keys in terminal and keeps credentials off disk.

IAM & Billing Permissions

Cost Explorer Access Guide

If you see an IAM Billing warning during execution, follow these two quick steps to enable access.

By default, IAM users cannot view billing data even with Administrator permissions until enabled by the account owner:

  1. Sign in to the AWS Management Console as the Root User (using primary email).
  2. Open the AWS Account Settings Page.
  3. Scroll down to IAM User and Role Access to Billing Information and click Edit.
  4. Check the box for Activate IAM Access and click Update.

Attach the managed AWS billing read policy to your IAM user or execution role:

  1. Open the IAM Users Console.
  2. Select your target user/role, click Add Permissions > Attach policies directly.
  3. Search for policy AWSBillingReadOnlyAccess and click Attach Policy.
Note: If Cost Explorer shows DataUnavailableException, AWS is still building initial billing records (takes up to 24 hours). The script automatically proceeds with resource scanning regardless!
